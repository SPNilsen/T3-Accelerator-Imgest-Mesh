Glossary
========

| **Term**                     | **Definition**                                                                                                                                             |
|------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Algorithm**                | A set of rules or instructions designed to perform a specific task. In machine learning, algorithms are used to find patterns in data or make predictions.  |
| **Annotation**               | Metadata added to data (e.g., labels, bounding boxes) to provide ground truth for supervised learning.                                                      |
| **Artificial Intelligence (AI)** | The simulation of human intelligence in machines that can perform tasks such as learning, reasoning, and self-correction.                                         |
| **Augmentation**             | Techniques used to increase the size and diversity of a dataset by applying transformations such as rotation, flipping, or scaling.                          |
| **AUC (Area Under Curve)**   | A performance metric for classification models, representing the area under the ROC curve and measuring the trade-off between true positives and false positives.|
| **Bias**                     | A systematic error introduced by the model, data, or process that can lead to inaccurate predictions or unfair results.                                       |
| **Bounding Box**             | A rectangular box used to define the location of an object in image annotations for object detection tasks.                                                 |
| **Classification**           | A type of predictive modeling where the output is a discrete label (e.g., defect vs. no defect).                                                           |
| **Clustering**               | An unsupervised learning method for grouping similar data points into clusters based on shared characteristics.                                              |
| **COCO Format**              | A widely used annotation format for object detection and segmentation tasks, organizing data into images, annotations, and categories.                        |
| **Convolutional Neural Network (CNN)** | A deep learning architecture particularly effective for image processing tasks, including classification and object detection.                                    |
| **Cross-Validation**         | A technique to evaluate model performance by dividing data into training and testing sets multiple times (e.g., K-fold cross-validation).                    |
| **Data Cleaning**            | The process of identifying and correcting errors or inconsistencies in data to improve its quality for analysis and modeling.                                |
| **Data Mining**              | The process of discovering patterns and insights in large datasets using statistical and machine learning techniques.                                        |
| **Data Preparation**         | The phase in the CRISP-DM methodology where raw data is cleaned, transformed, and organized for use in modeling.                                            |
| **Deep Learning**            | A subset of machine learning that uses neural networks with multiple layers to model complex patterns in data.                                              |
| **Defect Detection**         | The process of identifying anomalies or flaws in products, such as contact lenses, using automated systems.                                                 |
| **Defect-Specific Model**    | A machine learning model designed to detect and classify a specific type of defect, such as edge chips or center debris.                                     |
| **Edge Case**                | A scenario in data that is unusual or rare, often challenging for models to handle.                                                                         |
| **Evaluation**               | The phase in CRISP-DM where the performance of a model is assessed against defined metrics and business objectives.                                          |
| **Feature**                  | An individual measurable property or characteristic of data used as input for a machine learning model.                                                     |
| **Feature Engineering**      | The process of selecting, transforming, or creating features to improve model performance.                                                                 |
| **F1 Score**                 | A performance metric combining precision and recall, useful for imbalanced datasets.                                                                         |
| **False Positive (FP)**      | When a model incorrectly classifies a negative instance as positive.                                                                                       |
| **False Negative (FN)**      | When a model incorrectly classifies a positive instance as negative.                                                                                       |
| **Hyperparameter Tuning**    | The process of optimizing model hyperparameters to improve performance.                                                                                     |
| **Imbalanced Data**          | A dataset where certain classes (e.g., defective vs. non-defective lenses) are significantly underrepresented compared to others.                           |
| **Inference**                | The process of applying a trained model to new, unseen data to make predictions.                                                                            |
| **K-fold Cross-Validation**  | A validation technique that splits data into `k` subsets, using `k-1` for training and 1 for testing, repeated `k` times.                                    |
| **Label**                    | The ground truth or target value assigned to data during supervised learning.                                                                               |
| **Loss Function**            | A mathematical function used to evaluate the difference between predicted and actual values during model training.                                           |
| **Machine Learning (ML)**    | A branch of AI that focuses on creating algorithms that learn patterns in data and improve performance over time.                                            |
| **Metadata**                 | Additional information about data, such as annotations or attributes, used to enrich datasets for analysis.                                                |
| **Model**                    | A mathematical representation of a process or system trained using data to make predictions or decisions.                                                  |
| **Model Generalization**     | A model's ability to perform well on new, unseen data.                                                                                                      |
| **Model Overfitting**        | When a model performs well on training data but poorly on unseen data due to excessive complexity.                                                          |
| **Model Selection**          | The process of choosing the most suitable model or algorithm for a given problem based on evaluation metrics.                                               |
| **Neural Network**           | A set of algorithms modeled after the human brain, used to recognize patterns and relationships in data.                                                    |
| **Object Detection**         | A computer vision task that identifies and localizes objects within an image.                                                                               |
| **Precision**                | The proportion of true positives among all predicted positives, measuring the model's ability to avoid false positives.                                      |
| **Recall**                   | The proportion of true positives among all actual positives, measuring the model's ability to identify all relevant cases.                                   |
| **ROC Curve (Receiver Operating Characteristic)** | A graphical representation of a model's performance, showing the trade-off between true positive and false positive rates.                             |
| **Segmentation**             | A computer vision task that assigns a class label to every pixel in an image, dividing it into meaningful regions.                                           |
| **Supervised Learning**      | A type of machine learning where the model is trained on labeled data to make predictions.                                                                  |
| **True Positive (TP)**       | When a model correctly classifies a positive instance.                                                                                                      |
| **True Negative (TN)**       | When a model correctly classifies a negative instance.                                                                                                      |
| **Underfitting**             | When a model is too simple to capture patterns in the data, leading to poor performance on both training and unseen data.                                    |
| **Validation Set**           | A subset of data used to evaluate model performance during training but not used to train the model.                                                        |
| **Weighted Loss Function**   | A loss function that assigns different weights to classes, often used for imbalanced datasets.                                                              |
| **YOLO (You Only Look Once)**| A fast and efficient object detection algorithm designed for real-time applications.                                                                         |
| **Zero-Shot Learning**       | A type of machine learning that can recognize and classify objects it has not seen during training.                                                         |
